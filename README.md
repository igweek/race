# 班级赛车 · 任务争霸赛

Classroom Racing Game for Task-Based Learning

---

## 项目简介

“班级赛车 · 任务争霸赛”是一款面向课堂教学场景设计的轻量级互动竞赛小游戏。项目通过将课堂任务、答题表现和学习进度可视化为赛车竞速过程，增强课堂互动性，提升学生参与度与学习积极性。

本项目主要服务于任务驱动式教学、项目化教学及职业教育课堂，支持教师在课堂中进行实时加分、扣分与小组排名展示，适合投屏使用。

---

## 项目背景

在实际教学过程中，学生参与程度不均、课堂互动氛围不足是普遍存在的问题。传统口头加分或板书记录方式，反馈滞后、表现形式单一，不利于持续激励学生。

本项目以“竞赛化”和“游戏化”为设计思路，将教学过程中的评价行为转化为可视化、可感知的竞速过程，使学生能够直观感受到自身与小组的学习进度与课堂表现，从而提高课堂专注度与学习动力。

## 项目定位

* 项目类型：课堂互动竞赛工具（Web）
* 使用对象：教师、课堂教学演示
* 使用环境：浏览器（无需后端）
* 适用场景：

  * 分组任务竞赛
  * 课堂答题与抢答
  * 项目进度对比展示
  * 课堂表现加分与扣分管理

---

## 功能特性

* 小组竞速展示
  每个小组对应一条独立赛道，通过赛车位置展示任务完成进度。

* 任务步数可配置
  教师可根据课堂内容自由设置任务总数，用于控制比赛节奏。

* 实时积分榜
  支持积分实时统计与排序，前三名自动高亮显示。

* 冠军判定与奖励
  首个完成全部任务的小组自动获胜，并可触发随机奖励积分机制。

* 自适应界面布局
  根据小组数量自动调整赛道和积分榜布局，适合不同班级规模。

## 游戏玩法说明

### 1. 赛前设置

教师在右侧设置面板中完成以下操作：

1. 设置任务总数（1–99）
2. 输入小组名称（使用逗号分隔）
3. 点击“重置 / 开始比赛”，初始化比赛状态

---

### 2. 课堂竞赛过程

教师根据课堂表现进行操作：

| 操作方式   | 功能说明         |
| ------ | ------------ |
| 左键点击赛车 | 小组前进一步，积分加 1 |
| 右键点击赛车 | 小组后退一步，积分减 1 |
| 积分榜    | 实时显示各小组积分与排名 |

该操作方式适用于课堂答题、任务完成、纪律管理等多种教学情境。

---

### 3. 冠军与奖励

当某一小组率先完成全部任务时：

* 系统自动判定该小组为获胜者
* 弹出奖励界面
* 教师可引导抽取随机奖励积分
* 奖励积分自动计入总积分
